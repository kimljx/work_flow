# 系统架构说明

本项目采用单仓前后端分离结构：

- `frontend/`：Vue3 + Vite 管理端与成员端
- `backend/`：FastAPI 接口、调度器占位、邮件/QAX 集成边界
- `deploy/`：Windows、Win7、Linux、Docker 与离线部署资源
- `docs/`：架构、接口、部署、排障文档

## 核心业务域

- 任务域：维护任务基础信息、成员、里程碑、状态流转与起止时间
- 通知域：按邮件与即时消息两个渠道发通知，并拆分到成员维度记录送达、已读、重试情况
- 邮件域：负责邮箱拉取、模板匹配、动作解析、处理结果回写与手动收件
- 延期审批域：负责成员发起延期、管理员审批、审批幂等与版本控制
- 审计域：记录用户、任务、模板、提醒和审批等关键动作

## 核心规则

- 任务域与通知域保持解耦
- 邮件回复可以驱动任务状态变更
- 即时消息仅反馈通知送达与已读状态
- 延期审批通过原子更新与幂等键保证“首个有效生效”

## 最近界面与接口约定

- 任务列表必须展示开始时间、结束时间、状态色和优先级色
- 通知中心必须支持通知详情页，并展示通知正文和按成员拆分的送达、已读、重试信息
- 邮件列表只展示已匹配模板的邮件，并提供详情页、手动收件入口和下次自动收件倒计时
- 延期审批页使用工作台式卡片布局，展示统计指标、申请摘要和快速审批动作
