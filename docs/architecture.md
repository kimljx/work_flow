# 系统架构说明

本项目采用单仓前后端分离结构：

- `frontend/`：Vue 3 + Vite，包含管理员端和成员端页面。
- `backend/`：FastAPI 接口、任务通知、邮件收件、延期审批、导入导出和审计逻辑。
- `docs/`：接口、架构、部署与维护文档。

## 核心业务域

- 任务域：维护任务基础信息、成员、里程碑、子任务、状态流转与起止时间。
- 任务域中的子任务状态与邮件回复联动，成员邮件回执优先推进本人名下子任务，再反映到主任务状态。
- 任务列表接口直接返回子任务状态汇总，供工作台列表页快速判断执行进度。
- 通知域：按邮件和即时消息两个渠道发通知，并按成员维度记录送达、已读、重试和成员专属正文。
- 邮件域：负责邮箱拉取、模板匹配、回复动作解析、处理结果回写与手动收件。
- 延期审批域：负责成员发起延期、管理员审批、幂等校验与版本控制。
- 导入导出域：负责 Excel 模板、任务批量导入、导入历史和重复导入预警。
- 审计域：记录关键管理动作。

## 当前默认约定

- 登录页独立于主业务壳层渲染，左侧导航固定。
- 任务创建遵循“先选负责人和参与人员，再拆分子任务”。
- 任务创建界面已升级为“成员卡片内直接分配子任务”。
- 任务工作台支持按子任务状态进行快捷筛选，并可与延期、锁定、送达不足等条件组合筛选，优先服务执行视角的排查与跟进。
- 通知模板默认包含负责人、任务创建人和接收人对应子任务。
- 模板变量支持在前端以拖拽模块方式插入正文，并提供插入反馈，减少手写占位符时的拼写错误。
- 导入模板统一使用姓名字段，不使用用户名。
- 导入历史会保留失败摘要和重叠样例，供二次确认和事后追溯。
