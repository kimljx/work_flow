# 前端说明

- 技术栈：Vue 3 + Vite + Vue Router + Pinia
- 登录页独立于主业务壳层展示，未登录状态不显示左侧导航。
- 左侧导航栏固定在页面左侧，内容区滚动时导航不跟随上下滑动。

## 页面约定

- 任务创建页默认使用“当天 09:00 开始、当天 18:00 结束”的时间输入，并统一使用 `datetime-local` 录入任务时间。
- 任务创建页将“参与人员选择”和“子任务分配”合并为同一模块：先通过按钮选择负责人和参与成员，再在成员卡片内继续分配子任务。
- 任务列表页展示子任务状态汇总，并支持主状态、子任务状态、延期、锁定、送达不足等组合筛选。
- 任务详情页采用工作台式排版：顶部展示任务标题、进度条、关键指标和任务摘要，中间独立展示任务操作区，主体区域分为任务说明、里程碑、子任务、通知记录、延期记录、通知预览和状态时间线。
- 任务详情页不再单独展示“成员信息卡片”，成员信息收纳到头部摘要区，减少空间浪费。
- 任务详情页中的里程碑区采用“摘要卡 + 节点清单”的混合布局，不再和子任务区做生硬的等宽对半切分，优先提升空间利用率。
- 任务详情页的计划时长和实际时长统一按“天”展示，不再使用分钟或小时作为主展示单位。
- 任务详情页内置“邮件 / 即时消息预览”，管理员可切换渠道、通知类型和接收人，直接核对负责人、任务创建人和接收人子任务变量的最终渲染结果。
- 通知详情页除通知正文外，还会展示每位成员实际收到的个性化通知内容。
- 模板管理页展示当前模板可用占位符，并支持点击或拖拽变量插入正文，同时提供实时预览。
- 导入导出页支持下载 Excel 模板、上传导入、查看导入历史，以及高重叠导入时的二次确认。
- 详情页、编辑页、通知详情页等返回逻辑统一遵循“从哪进返回哪”。

## 公共层约定

- `src/api/http.js` 统一处理接口前缀、令牌注入、401 自动刷新和全局等待状态。
- `src/stores/auth.js` 统一管理登录态、持久化和当前用户资料。
- `src/stores/loading.js` 维护全局等待计数，避免重复点击和重复提交。
- `src/router/index.js` 统一做登录校验与角色路由守卫。
- `src/utils/format.js` 统一处理日期、时间、时长和倒计时格式化。
- `src/constants/taskUi.js` 统一维护任务状态、子任务状态和优先级的颜色映射。
