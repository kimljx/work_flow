# 系统使用手册

## 1. 角色说明

系统主要分为两类角色：

- 管理员：创建任务、配置模板、查看通知、管理邮件、处理延期审批
- 成员：查看分配给自己的任务、接收通知、通过邮件回执更新任务状态

## 2. 登录

启动系统后，在浏览器访问：

```text
http://部署电脑IP:8000/
```

默认账号：

- 管理员：`admin`
- 成员：`member`

默认密码由 `app\.env` 中的 `DEFAULT_PASSWORD` 决定。

## 3. 管理员日常操作

### 3.1 创建任务

进入任务模块后可新建任务，建议按以下顺序填写：

1. 任务标题
2. 任务内容
3. 负责人
4. 参与人员
5. 开始时间与结束时间
6. 里程碑
7. 子任务
8. 到期提醒与备注

说明：

- 新建任务默认开始时间为当天 09:00，结束时间为当天 18:00
- 子任务可按参与成员逐个分配
- 保存后系统会按模板生成通知预览并发送通知

### 3.2 查看任务详情

任务详情页可以查看：

- 任务概览
- 计划时长与实际时长
- 里程碑进度
- 子任务状态
- 通知记录
- 延期记录
- 邮件 / 即时消息预览
- 状态时间线

### 3.3 编辑任务

在任务详情页点击“编辑任务”即可修改。

返回逻辑遵循：

- 从哪进入编辑页，就返回到哪里

### 3.4 手动提醒

在任务详情页可以点击“手动提醒”，系统会按当前模板再次生成并发送通知。

### 3.5 导入导出

进入导入导出页可完成：

- 下载任务 Excel 模板
- 上传批量导入任务
- 查看导入历史
- 查看重复导入预警

导入模板中用户字段统一使用“姓名”，不是用户名。

## 4. 通知与模板

### 4.1 通知中心

通知中心可查看：

- 通知渠道
- 通知类型
- 送达情况
- 已读情况
- 每位成员的个性化通知快照

### 4.2 模板管理

模板管理支持：

- 维护邮件 / 即时消息模板
- 查看变量占位符
- 点击或拖拽变量插入正文
- 实时预览模板内容

任务通知模板会自动带入：

- 任务名称
- 负责人
- 任务创建人
- 当前接收人的子任务内容

## 5. 邮件模块

邮件模块主要用于：

- 查看匹配到模板的邮件
- 查看邮件详情
- 手动收取邮件
- 查看下次自动收取倒计时

系统支持按邮件回复推进任务状态。

常见回复方式：

- `进行中 + 备注`
- `已完成 + 备注`
- `延期 + 新日期 + 原因`

## 6. 延期审批

管理员可在延期审批页查看：

- 待审批申请
- 申请人
- 原截止日期
- 申请截止日期
- 申请原因
- 批量审批

任务详情页中的延期记录会同步展示审批结果。

## 7. 成员日常操作

成员登录后主要关注：

- 我的任务
- 我的通知
- 分配给我的子任务
- 延期申请结果

成员可在任务详情中查看：

- 主任务要求
- 自己负责的子任务
- 通知记录
- 状态变化

## 8. 常见使用建议

- 创建任务后先看一次“邮件 / 即时消息预览”，确认模板渲染无误再批量推进。
- 如果暂时不启用邮箱，先将 `MAIL_AUTO_POLL_ENABLED=false`，避免日志中持续报收件配置错误。
- 升级系统前先备份 `app\.env` 和 `app\backend\data\app.db`。
